# CLAUDE.md — beanstalkd Codebase Guide

This file provides AI assistants with the context needed to work effectively in the beanstalkd codebase.

## Project Overview

**beanstalkd** is a simple, fast, general-purpose work queue server written in C. It exposes a TCP-based ASCII protocol (the "Beanstalk protocol") for producers to enqueue jobs and consumers to reserve and process them. It is a mature, production-grade project focused on stability, performance, and correctness over new features.

- **Language**: C (C99 required)
- **Latest version**: 1.13 (2023-03-12)
- **Supported platforms**: Linux 2.6.17+, macOS, FreeBSD, Solaris/Illumos
- **Architecture**: Single-threaded, event-driven (epoll on Linux, kqueue on macOS/BSD)
- **Default port**: 11300

## Build System

The project uses a plain **GNU Makefile** — no autoconf, no CMake.

### Common Commands

```sh
make                        # Build the beanstalkd binary
make check                  # Build and run all tests
make bench                  # Run benchmarks
make install                # Install to /usr/local/bin
make install PREFIX=/usr    # Install with custom prefix
make clean                  # Remove build artifacts
make CC=clang               # Build with clang
make CFLAGS=-O2             # Optimized build
```

### Build Variables

| Variable | Default | Description |
|---|---|---|
| `PREFIX` | `/usr/local` | Installation prefix |
| `CC` | system default | C compiler (gcc or clang) |
| `CFLAGS` | _(appended)_ | Additional compiler flags |
| `OS` | `$(uname \| tr A-Z a-z)` | Platform selector (`linux`, `darwin`, `freebsd`, `sunos`) |
| `USE_SYSTEMD` | auto-detect | `yes`/`no`/auto — enables systemd socket activation |

**Compiler flags always applied**: `-Wall -Werror -Wformat=2 -g`

### Platform-Specific Linking

- **Linux**: links `-lrt`
- **Solaris**: links `-lxnet -lsocket -lnsl`
- **macOS/BSD**: no extra libs
- **systemd** (Linux, optional): links `libsystemd` if found via `pkg-config`

### Version Generation

`vers.c` is auto-generated by `verc.sh` from git tags — **never edit it manually**. It is regenerated automatically when the git-derived version changes.

### FreeBSD Note

`freebsd.o` is compiled from `darwin.c` (see Makefile line 122: `freebsd.o: darwin.c`). They share the same kqueue-based implementation.

## Central Header: `dat.h`

`dat.h` is the single header that defines **all** types, constants, macros, and `extern` declarations used across the project. Every `.c` file implicitly depends on it (Makefile rule: `$(OFILES): $(HFILES)`).

## Coding Conventions

### Naming

| Entity | Convention | Example |
|---|---|---|
| Types / Structs | `CamelCase` | `Job`, `Tube`, `Conn`, `Jobrec` |
| Functions | `snake_case` or `moduleverb` | `make_job()`, `job_find()`, `heapinsert()` |
| Macros / Constants | `UPPER_SNAKE_CASE` | `MAX_TUBE_NAME_LEN`, `UNUSED_PARAMETER` |
| Module-prefixed functions | `module_verb()` | `job_free()`, `tube_find()`, `ms_append()` |
| Count fields | `_ct` suffix | `reserve_ct`, `timeout_ct` |
| Function pointer types | `_fn` suffix | `less_fn`, `setpos_fn`, `ms_event_fn` |

### Style

- **Standard**: C99
- **Compiler enforcement**: `-Wall -Werror -Wformat=2` — all warnings are errors
- Avoid global mutable state; the only global is `srv` (the single `Server` instance)
- Use reference counting for resource management (tubes, WAL files)
- Prefer callback-based generic data structures (heap, multiset)
- Mark unused function parameters with `UNUSED_PARAMETER(x)` macro

### Memory Management

```c
zalloc(n)    // malloc + memset(0) — preferred for structs
new(T)       // expands to zalloc(sizeof(T))
free(p)      // direct free
```

Return `0` / `NULL` on allocation failure — callers must check.

### Error Reporting

```c
twarn("message")          // logs: progname: file:line in func: message: strerror(errno)
twarnx("message")         // same without errno suffix
```

Exit codes are defined in `main.c`. Fatal errors call `exit()` directly.


## Job State Machine

```
          put with delay
  +--------------+
  |              |
  |  [DELAYED]   |<--------------------------+
  |              |                           |
  +--------------+  (time passes)            | release with delay
         |                                   |
         | deadline reached                  |
         v                                   |
  +--------------+    reserve    +-----------+--------+
  |              |-------------->|                    |
  |   [READY]    |               |    [RESERVED]      |
  |              |<--------------+                    |
  +--------------+    release    +---------+----------+
                                           |
                                    bury   |   delete
                                     |     |
                                     v     v
                              +-----------+   *poof*
                              |           |
                              |  [BURIED]  |
                              |           |
                              +-----------+
                                    |
                              kick  |   delete
                                    v     v
                                 [READY]  *poof*
```

Jobs also have a TTR (time-to-run) timer: if a reserved job is not deleted/released/buried before TTR expires, it returns to `READY`.

## Protocol Summary

beanstalkd uses a **TCP, line-oriented ASCII protocol** with binary job bodies. See `doc/protocol.txt` for the full specification.

### Producer Side

```
use <tube>\r\n                              → USING <tube>\r\n
put <pri> <delay> <ttr> <bytes>\r\n
<data>\r\n                                 → INSERTED <id>\r\n
```

### Consumer Side

```
watch <tube>\r\n                           → WATCHING <count>\r\n
ignore <tube>\r\n                          → WATCHING <count>\r\n
reserve\r\n                                → RESERVED <id> <bytes>\r\n<data>\r\n
reserve-with-timeout <seconds>\r\n         → RESERVED / TIMED_OUT / DEADLINE_SOON
reserve-job <id>\r\n                       → RESERVED <id> <bytes>\r\n<data>\r\n
delete <id>\r\n                            → DELETED\r\n
release <id> <pri> <delay>\r\n             → RELEASED\r\n
bury <id> <pri>\r\n                        → BURIED\r\n
touch <id>\r\n                             → TOUCHED\r\n
kick <bound>\r\n                           → KICKED <count>\r\n
kick-job <id>\r\n                          → KICKED\r\n
```

### Inspection

```
peek <id>\r\n                              → FOUND <id> <bytes>\r\n<data>\r\n
peek-ready\r\n / peek-delayed\r\n / peek-buried\r\n
stats\r\n / stats-job <id>\r\n / stats-tube <tube>\r\n
list-tubes\r\n / list-tubes-watched\r\n / list-tube-used\r\n
pause-tube <tube> <delay>\r\n              → PAUSED\r\n
quit\r\n
```

### Protocol Limits

- Command line: max 224 bytes (including `\r\n`)
- Tube name: max 200 bytes
- Job body: default 65535 bytes; configurable up to 1 GB via `-z`
- Job ID: 64-bit unsigned integer
- Priority: 32-bit unsigned (0 = highest priority)
- Delay / TTR: 32-bit unsigned (seconds)

## Write-Ahead Log (Binlog / WAL)

When started with `-b DIR`, beanstalkd persists jobs to a binary write-ahead log:

- Segment files: `DIR/binlog.1`, `DIR/binlog.2`, … (current format version: `Walver = 7`)
- Each segment has a max size (default 10 MB, configurable with `-s`)
- Garbage collection (`walg.c`): rewrites segments keeping only live jobs
- fsync policy: configurable via `-f MS` (default 50 ms) or `-F` (never)
- The `Jobrec` struct is the on-disk record format; changing it requires a version bump

## Key Algorithms

### Job Hash Table (`job.c`)

- Open addressing with linear probing
- Resize using prime capacities from `primes.c`
- Scale up when `used > cap << 2`; scale down when `used < cap >> 4`
- Hash: `job_id % capacity`

### Binary Min-Heap (`heap.c`)

- Generic: accepts `less_fn` and `setpos_fn` callbacks
- Used for: ready queue (ordered by priority, then job ID), delayed queue (ordered by deadline)
- Each job stores its heap position for O(log n) deletion

### Event Loop (`serv.c`, `linux.c`, `darwin.c`, `sunos.c`)

- Single-threaded, non-blocking I/O
- Platform abstraction: `Socket` struct with `readwant`, `wantr`, `wantw` fields
- Linux: `epoll_create` / `epoll_ctl` / `epoll_wait`
- macOS/BSD: `kqueue` / `kevent`

## Development Guidelines

From `CONTRIBUTING.md`:

1. **Priority order**: stability > bug fixing > clarity > performance > new features
2. **New features are rarely accepted** — this is a mature project
3. **Keep PRs focused**: one topic per PR; avoid unrelated whitespace/comment edits
4. **Discuss large changes first** (mailing list) before investing time
5. **Commit messages**: follow the [Go contributing guide style](https://golang.org/doc/contribute.html#commit_messages) — short subject line, blank line, detailed body if needed
6. **C99 compliance required** — no compiler extensions

### What Not to Change Without Strong Justification

- Protocol wire format (breaks all clients)
- `Jobrec` on-disk layout (requires WAL version bump)
- Single-threaded architecture (fundamental design decision)
- Default port 11300 (widely deployed)

## Common Workflows

### Adding a New Protocol Command

1. Add parsing in `prot.c` — find the command dispatch table
2. Add the handler function in `prot.c`
3. Update `doc/protocol.txt` with the new command spec
4. Add integration tests in `testserv.c`
5. Update `doc/beanstalkd.ronn` if the man page is affected

### Adding a New Data Structure or Utility

1. Implement in a new `.c` file (or add to `util.c` for small utilities)
2. Declare the interface in `dat.h`
3. Add the `.o` file to `OFILES` in `Makefile`
4. Write tests in an appropriate `test*.c` file

### Debugging

```sh
./beanstalkd -VVV          # Maximum verbosity
./beanstalkd -VVV -b /tmp  # With binlog in /tmp
```

Use `twarn()` / `twarnx()` (defined via `util.c`) for diagnostic output — these respect `stderr` and include file/line context automatically.

### Code Coverage

The project uses `codecov.io`. Coverage is collected by CI (`.travis.yml`, `.github/`). To build with coverage locally:

```sh
make CFLAGS="--coverage" LDFLAGS="--coverage" check
```
